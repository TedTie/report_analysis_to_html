# Quality Checklist

## Data Insight

- Data files and versions are identified.
- Time period is clear.
- Incomplete period or cutoff dates are marked when applicable.
- Calculation logic is consistent.
- Old and new data versions are not mixed.
- Missing data is not treated as complete data.
- Metrics are not analyzed beyond the user's requested scope without reason.
- Ambiguous business terms are confirmed or explicitly listed as assumptions.

## Management Report

- The report follows the confirmed flow.
- The tone fits the user and audience.
- Management judgments are supported by data and source documents.
- Future plans are actionable, not slogans.
- Repeated insights are reduced.
- Internal prompts, process notes, and production comments are removed.
- Personal voice is preserved when the user asks for it.
- Claims do not exceed the evidence.

## HTML Report

- HTML matches confirmed `report.md`.
- Data values match source report and data insights.
- Text is readable and not overcrowded.
- No white-on-white or low-contrast text.
- No element overlaps or hides content.
- Tables and charts are aligned and readable.
- Decorative assets do not distract from content.
- Image paths are local and valid.
- No external dependency is required unless explicitly approved.
- The final ZIP opens from the root-level HTML file after extraction.
